# Logic Bomb

"The Digital Time-Bomb" ,malicious code that sits perfectly still and silent for months, only "exploding" and wiping your data when a specific condition is triggered.

A **Logic Bomb** is a patient assassin. Unlike a **[[virus|Virus]]** that immediately starts infecting every file it touches, a logic bomb waits. It might be programmed to wait until the company reaches its 1,000th customer, or until a specific employee's name is deleted from the HR system. Once that "Logic" condition is met, the bomb goes off ,deleting the database or crashing the server.

Think of it like **A "Dead Man's Switch"**:
*   An angry IT admin plants code that says: *"If I don't log in at least once every 7 days, delete everything."*
*   If that admin gets fired, they can't log in.
*   One week later, the logic bomb "explodes," and the company loses its data. 

## Key Characteristics
1.  **The Inside Job:** Logic bombs are almost always planted by people who already have legitimate access ,like disgruntled employees or contractors who haven't been paid.
2.  **Extreme Stealth:** Because the code isn't "doing anything" while it's waiting, it is incredibly hard for security scanners to find. It just looks like a boring, unused piece of standard software.
3.  **Maximum Pain:** The goal is rarely to steal money; it’s usually to inflict as much chaos as possible as a form of revenge.

## FAQs

*1. Is it a type of malware?*
**Yes.** But it's a "Bespoke" breed. Most malware is designed to spread (like a virus). A logic bomb is custom-written for one specific target and doesn't try to go anywhere else.

*2. How can you find one?*
**Code Reviews.** Because these bombs are hidden in the literal lines of a company's software, the only way to catch them is for other developers to read through the code and ask: *"What does this weird, unused piece of logic actually do?"*
